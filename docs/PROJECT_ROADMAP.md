## 開発方針

### 基本原則
1. **段階的改善**: まず動くものを作り、徐々に高度化
2. **後方互換性**: 既存データを壊さない
3. **非破壊的変更**: featureブランチで開発
4. **可観測性**: エラーの理由を明確に表示

### 技術スタック
- Frontend: React + TypeScript + Vite
- UI: ドラッグ&ドロップ（react-dnd）
- Export: Excel（xlsx）、PDF（jspdf）
- AI: 将来的に自然言語処理導入予定
